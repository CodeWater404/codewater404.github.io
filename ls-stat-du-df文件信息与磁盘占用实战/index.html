ls、stat、du、df 实战：文件信息与磁盘占用怎么查

发表于2026-07-14|更新于2026-07-14|tools
|总字数:1.7k|阅读时长:5分钟|浏览量:

服务器磁盘报警 90%，挨个目录 ls -la 看着都不大，就是找不到空间被谁吃了——这是用错了工具：ls 根本不统计目录树的总占用。查文件和磁盘信息的这组命令，每个回答的问题不一样，对上号才查得动。

这是 Linux 命令系列的一篇，承接 Linux 常见命令与工具地图。

ls：列目录内容，先看懂 -la 的每一列

ls 列出目录内容，默认不显示 . 开头的隐藏文件、不显示详情，只给一排文件名。日常几乎都是配着参数用：

tree：树状展示目录层级

tree 以树状缩进展示目录结构，比 ls -R（递归列出但没有层级视觉）直观得多。它不是系统自带命令，macOS 和很多精简系统默认没有，要先装（brew install tree / apt install tree）：

file：不看扩展名，判断文件真实类型

file 读取文件开头的魔数（magic number）来判断真实类型，完全不依赖扩展名——文件没有后缀、或者怀疑后缀是假的（比如伪装成 .jpg 的脚本）时用它：

stat：文件元信息与三个时间戳

stat 显示文件的完整元信息：大小、权限、inode、属主，以及三个容易搞混的时间戳。这三个时间戳的区别值得单独记住：

最容易讲错的是 ctime：它不是”创建时间”（create），是”变更时间”（change）——指的是元信息的变更。给文件 chmod 改个权限，内容一个字节没动，mtime 不变，但 ctime 会更新。排查”文件内容没变但权限什么时候被人改过”，看的就是 ctime。

注意 stat 是 BSD/GNU 差异比较大的命令：macOS 上直接 stat file 输出一行紧凑格式，要 -x 才有上面这种可读输出；Linux 上正相反，直接 stat file 就是完整格式。跨平台写脚本时别依赖它的输出格式。

du 和 df：一个查”谁占的”，一个查”还剩多少”

这两个命令名字长得像，回答的问题完全不同：df（disk free）查文件系统级别的使用率——“这块盘还剩多少”；du（disk usage）统计某个目录树的实际占用——“这个目录一共吃了多少”。磁盘满排查时两个要配合用：

一个偶尔会遇到的现象：du 把整个分区加起来的数字比 df 显示的已用空间小很多。常见原因是有文件已经被删除、但还有进程占着它的文件句柄——这种文件 du 扫不到（目录里已经没有它了），df 却依然把它算在已用空间里，重启对应进程空间才会真正释放。

组合实战：磁盘满了怎么一步步定位

从分区到目录再到具体文件，三四层下钻通常就能定位。日志文件撑满磁盘是最常见的答案，所以很多人直接从 du -sh /var/log 查起。

常用参数速查表

命令/参数 作用
ls -la 长格式列出全部文件（含隐藏）
ls -lt / ls -lS 按修改时间 / 按大小排序
tree -L N 树状展示目录，限制 N 层深度
tree -d 只显示目录
file 文件 按内容魔数判断文件真实类型
stat 文件 查看元信息；macOS 加 -x 才是可读格式
atime / mtime / ctime 最后访问 / 内容修改 / 元信息变更时间
df -h 各分区磁盘使用率
du -sh 目录 目录树总占用
du -sh /* | sort -rh 磁盘满排查：按占用从大到小排

这组命令按问题分工：文件列表和排序找 ls，目录结构找 tree，文件类型找 file，元信息和时间戳找 stat，磁盘空间 df 定分区、du 定目录。磁盘报警时别再挨个 ls 了——df 加 du 逐层下钻，几条命令就能揪出占空间的元凶。
